# 판교테크노밸리·위례신도시 업무지구 비교분석 대시보드

## 1. 프로젝트 개요

이 프로젝트는 판교테크노밸리와 위례신도시 업무지구를 공공데이터 기반으로 비교하는 정적 웹 대시보드이다. 토지이용, 건축물 용도, 대중교통 접근성, 인구·종사자 지표를 한 화면에서 비교할 수 있도록 구성하였다.

## 2. 배포 URL

GitHub Pages URL: https://jung-ain.github.io/smartcity-final/

## 3. 주요 기능

- 두 지역 경계 및 토지이용/용도지역 지도 시각화
- 30분/60분 등시간권 레이어
- 도달가능 인구·종사자 수 표시
- 판교·위례 비교 통계 패널
- 필지/건축물 클릭 시 속성 조회
- 버스·지하철 접근성 슬라이더 기반 지도 표현

## 4. 실행 방법

```bash
npm install
npm run dev
npm run build
npm run preview
```

## 5. 데이터 구성

대시보드 구동에 필요한 최종 정적 데이터만 `public/data/`에 포함하였다. 원본 SHP, 원본 CSV, API 원본 응답, 중간 산출물, 임시 파일은 저장소에 포함하지 않는다. 실제 사용 파일 목록은 `docs/used_data_inventory.md`에 정리하였다.

## 6. 사용 데이터 출처

- LH 공공택지정보시스템: 택지지구 경계 및 토지이용계획도
- 브이월드: 연속지적도, GIS 건물정보, Geocoder API
- 국토교통부 건축HUB: 건축물대장 표제부
- 통계청 SGIS: 인구·사업체·종사자 통계 및 격자경계
- 서울 열린데이터광장: 서울 버스정류소, 지하철 관련 기초자료
- 경기데이터드림: 경기 버스정류소
- OpenStreetMap: 도로망 및 배경지도
- 수업 제공자료: 수도권 지하철 네트워크

## 7. 기준연도

- 인구·종사자 및 사업체 지표: SGIS 기반 2023년 자료를 중심으로 산정
- 토지이용계획 및 경계: 분석 시점에 확보한 VWorld/LH 기반 자료 사용
- 건축물대장 및 GIS 건물정보: 건축HUB·VWorld에서 확보한 최신 가용 자료 사용
- 대중교통 접근성: 제공 지하철 네트워크와 수집 버스 정류장 자료를 전처리하여 산정

데이터별 세부 출처와 처리 기준은 `docs/data_sources.md`를 참고한다.

## 8. 재현성

전처리 스크립트는 `scripts/` 폴더에 포함한다. 원자료는 용량 문제와 라이선스 문제로 저장소에 포함하지 않으며, README와 `docs/data_sources.md`에 출처를 명시하였다.

## 9. 한계

- 일부 데이터의 기준연도와 기준월이 서로 다르다.
- 일부 건축물 속성은 결측값이 존재할 수 있다.
- 등시간권은 전처리된 네트워크 및 통계자료를 기반으로 산정되었다.
- GitHub Pages 정적 배포를 위해 원자료와 중간 산출물은 포함하지 않는다.
